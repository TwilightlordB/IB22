Работа с ssh
1. Установление доверенных отношений с сервером 
При первом соединении с сервером клиент получает публичный ключ сервера и пользователь должен потвердить этого ключа
Сличение ключа можно провести с помощью сличения отпечатка (fingerprint)
После подтверждения соединения на стороне клиента, информация о публичном ключе сервера записывается в файл в домашней папке пользователя ~/.ssh/know_hosts
При удалении этого файла доверительные соединения потребуется установить заново.

2. Генерация ssh-ключей пользователем
ssh-keygen -t <тип алгоритма шифорования> -C <электронная почта владельца ключа>
ssh-keygen -t ed25519 -C "TwilightlordB@yandex.ru"
Для двухфакторной защиты секретного ключа необходимо использовать парольную фразу (длиной более 12 символов)

3. Копирование публичного ключа пользователя на сервер 
ssh-copy-id <имя пользователя на сервере>@<адрес сервера>
ssh-copy-id student@172.16.7.85
Ввести пароль от удаленой учетной записи

4. Добавление секретного ключа в клиента ssh
ssh-agent bash
ssh-add <путь к файлу секретного ключа>
ssh-add ~/.ssh/id_ed25519
Ввести парольную фразу для однократного получения доступа к секретному ключю

5. После выполнения шагов со второго по пятый соединение с сервером будет осуществляться без пароля
(аунтефикация и идентификация проходит при помощи пары ключей секретный-публичный)
